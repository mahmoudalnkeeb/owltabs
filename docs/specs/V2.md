## Part 1 — Product Overview

### What it is

A Manifest V3 browser extension that replaces the new tab page in Brave (and Chrome). It is a **client-side-only** productivity dashboard. No server, no backend, no proxies owned by the developer.

### Feature Set

| Feature                                                   | Priority |
| --------------------------------------------------------- | -------- |
| Centered search bar with keyboard shortcut                | P0       |
| Quick links grid (favicon + label, configurable)          | P0       |
| RSS feed grid (multi-source, cached locally)              | P0       |
| Feed category filter + in-feed search                     | P1       |
| Save articles to local reading list                       | P1       |
| Settings drawer (search engine, links, feeds, appearance) | P1       |
| AI assistant panel (Gemini, token-gated)                  | P2       |
| Keyboard navigation throughout                            | P1       |

### Non-Goals (v1.0)

- No server sync / account system
- No browser history integration
- No light mode (dark-first; system light mode note in Part 6)
- No drag-to-reorder (add/remove only in v1)

### 2.4 Typography Rules

Inherit all type scales from `tokens.css`. Usage map:

| Scale                        | Used for                                        |
| ---------------------------- | ----------------------------------------------- |
| `--text-h2` (26px 600)       | Section headings ("Your Feed")                  |
| `--text-h3` (19px 600)       | Article card titles                             |
| `--text-body` (15px)         | Settings rows, AI messages                      |
| `--text-body-sm` (13px)      | Article excerpts, feed source names, timestamps |
| `--text-caption` (11px mono) | Category labels, keyboard shortcut hints        |
| `--text-code` (13px mono)    | API key input, URL inputs in settings           |

### 2.5 Motion Rules

Use the existing motion tokens. Rules for this extension:

- **Page load**: sections fade up sequentially — hero (0ms), quick links (60ms), feed (120ms)
- **Omnibox focus expand**: `height` transition `var(--dur-base) var(--ease-out)` to reveal mode tabs
- **Panel slide-in**: `transform: translateX(100%)` → `translateX(0)`, `var(--dur-slow) var(--ease-out)`
- **Backdrop**: opacity `0 → 1`, `var(--dur-base)`
- **Card hover**: `box-shadow` and `border-color` change only — NO transform (avoid layout jank in a grid)
- **Feed card save**: icon scale pulse `1 → 1.2 → 1`, `var(--dur-fast)`
- **Skeleton shimmer**: horizontal gradient sweep, `1.8s ease-in-out infinite`
- **All animations**: respect `prefers-reduced-motion` — use the existing `@media` block in `tokens.css`

**Behavior:**

- Tab between Web/Feed pills with arrow keys while footer is visible
- On mobile / narrow: max-width is 100% of container minus `--sp-6` padding

## Part 5 — Keyboard Navigation

### Global Shortcuts

| Shortcut        | Action                                                         |
| --------------- | -------------------------------------------------------------- |
| `/` or `Space`  | Focus omnibox (when not already in an input)                   |
| `⌘K` / `Ctrl+K` | Focus omnibox (anywhere)                                       |
| `⌘,` / `Ctrl+,` | Toggle settings drawer                                         |
| `⌘/` / `Ctrl+/` | Toggle AI panel                                                |
| `Escape`        | Close active panel → blur search → clear search                |
| `Tab`           | Standard browser tab navigation                                |
| `1`–`9`         | Jump to quick link by index (when no panel open, not in input) |
| `F`             | Focus feed search (when not in input, no panel open)           |

### Shortcut Priority

When multiple shortcuts could fire: `Escape` always wins (closes/clears in layered order: AI panel → Settings → Feed search → Search input focus).

### Focus Management

- When a panel opens: focus the panel's first focusable element (close button or first input)
- When a panel closes: return focus to the element that triggered it (save `lastFocusedElement` before opening)
- Use `aria-live="polite"` on AI message log for screen reader announcements
- All interactive elements reachable via keyboard (`tabindex` not needed beyond native elements)

### Implementation (`keyboard.ts` module):

```typescript
// keyboard.ts
export function initKeyboard(handlers: KeyboardHandlers) {
  document.addEventListener("keydown", (e) => {
    const inInput = document.activeElement?.matches(
      "input, textarea, [contenteditable]",
    );

    if (e.key === "Escape") {
      handlers.escape();
      return;
    }
    if ((e.metaKey || e.ctrlKey) && e.key === "k") {
      e.preventDefault();
      handlers.focusSearch();
      return;
    }
    if ((e.metaKey || e.ctrlKey) && e.key === ",") {
      e.preventDefault();
      handlers.toggleSettings();
      return;
    }
    if ((e.metaKey || e.ctrlKey) && e.key === "/") {
      e.preventDefault();
      handlers.toggleAI();
      return;
    }

    if (!inInput) {
      if (e.key === "/" || e.key === " ") {
        e.preventDefault();
        handlers.focusSearch();
        return;
      }
      if (e.key === "f" || e.key === "F") {
        handlers.focusFeedSearch();
        return;
      }
      if (e.key >= "1" && e.key <= "9") {
        handlers.activateQuickLink(parseInt(e.key) - 1);
        return;
      }
    }
  });
}
```

### 6.2 RSS Service (`services/rss.ts`)

```typescript
const SEARCH_ENGINES: Record<string, string> = {
  brave: "https://search.brave.com/search?q={query}",
  google: "https://www.google.com/search?q={query}",
  ddg: "https://duckduckgo.com/?q={query}",
  bing: "https://www.bing.com/search?q={query}",
};

// Extension pages can fetch cross-origin if host_permissions includes *://*/*
async function fetchFeed(url: string): Promise<FeedItem[]> {
  const res = await fetch(url, { cache: "no-cache" });
  const text = await res.text();
  const xml = new DOMParser().parseFromString(text, "text/xml");

  // Detect RSS 2.0 vs Atom
  const isAtom = xml.querySelector("feed") !== null;
  return isAtom ? parseAtom(xml) : parseRSS(xml);
}

function parseRSS(xml: Document): FeedItem[] {
  return Array.from(xml.querySelectorAll("item")).map((item) => ({
    title: item.querySelector("title")?.textContent ?? "",
    url: item.querySelector("link")?.textContent ?? "",
    excerpt: stripHtml(
      item.querySelector("description")?.textContent ?? "",
    ).slice(0, 200),
    thumbnailUrl:
      item
        .querySelector('media\\:thumbnail, enclosure[type*="image"]')
        ?.getAttribute("url") ??
      extractOgImage(item.querySelector("description")?.textContent ?? ""),
    publishedAt: new Date(
      item.querySelector("pubDate")?.textContent ?? "",
    ).toISOString(),
    // ... remaining fields
  }));
}
```

**Cache strategy:**

1. On new tab open: serve from `chrome.storage.local` immediately (no flash of empty state)
2. If `Date.now() - cache.fetchedAt > refreshIntervalMs`: trigger background refresh
3. Background refresh: fetch all enabled feeds → merge → deduplicate by `id` → write back to storage → dispatch `CustomEvent('feed-updated')` that the page listens to

### 7.4 State Management Pattern

No framework. Use a lightweight reactive store:

```typescript
// store.ts
type Listener<T> = (val: T) => void;

export function createStore<T>(initial: T) {
  let state = initial;
  const listeners = new Set<Listener<T>>();
  return {
    get: () => state,
    set: (updater: (s: T) => T) => {
      state = updater(state);
      listeners.forEach((fn) => fn(state));
    },
    subscribe: (fn: Listener<T>) => {
      listeners.add(fn);
      return () => listeners.delete(fn);
    },
  };
}

// Usage:
export const feedStore = createStore<FeedItem[]>([]);
export const uiStore = createStore({
  aiPanelOpen: false,
  settingsOpen: false,
  activeFilter: "all",
  searchQuery: "",
});
```

Components subscribe to stores and re-render only their affected DOM nodes (no virtual DOM needed — the page is relatively small).

## Part 9 — Design Decisions & Rationale

These are final decisions — do not relitigate them:

| Decision                                | Rationale                                                                                                                                         |
| --------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------- |
| Dark-only in v1                         | Vesper is dark-first; adding light mode is a full second design pass, not a quick swap                                                            |
| No framework (Lit/React)                | The page is a single-session document with modest complexity; a reactive store + direct DOM is faster to ship and lighter to load                 |
| `chrome.storage.sync` for settings      | Settings sync across user's devices automatically; feed cache stays local (too large for sync)                                                    |
| `host_permissions: *` for RSS           | Extensions have this as a legitimate pattern for user-configured RSS sources on any domain; no proxy server needed                                |
| Gemini API key in `chrome.storage.sync` | Chrome's sync storage is encrypted at rest and in transit; it's the correct mechanism for user credentials in extensions                          |
| `DOMParser` for RSS                     | No library dependency; works in all Chromium versions; handles both RSS 2.0 and Atom 1.0                                                          |
| Vesper orange accent as primary         | Keep brand continuity from the existing design system; the warm orange on dark surface reads well and doesn't conflict with Brave's own UI chrome |
| Feed cards without transform hover      | `box-shadow` + `border-color` only — avoids layout shifts and compositing jank in a dense grid                                                    |

## Appendix B — Relative Timestamp

```typescript
export function relativeTime(iso: string): string {
  const diff = Date.now() - new Date(iso).getTime();
  const mins = Math.floor(diff / 60_000);
  if (mins < 1) return "just now";
  if (mins < 60) return `${mins}m ago`;
  const hrs = Math.floor(mins / 60);
  if (hrs < 24) return `${hrs}h ago`;
  const days = Math.floor(hrs / 24);
  if (days < 7) return `${days}d ago`;
  return new Date(iso).toLocaleDateString("en-US", {
    month: "short",
    day: "numeric",
  });
}
```
